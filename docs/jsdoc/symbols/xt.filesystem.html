Namespace xt.filesystem

The filesystem module for the xt framework in node.js is a collection of operations that make file io more convenient for extensions and objects.
Defined in: filesystem.js.

Splits a string into individual lines. Can be used for normal strings or from file data. This function is synchronous and returns an array of lines.

The string or file data to parse.

directoryFiles(path, callback, extension)
Find and return paths to all files in a specified directory. Optionally an extension can be specified as the third paramater. The second parameter is the callback on completion. This method is asynchronous.
Parameters:
{String} path
The path to the directory.
{Function} callback
The callback expects a single (array) parameters of the results.
{String} extension Optional
The file extension to look for (otherwise all file types).

{String} shorten(path, max)
Takes what might be a very long file path and shortens it to a (default 3) maximum number of slashes counting from the end and prepends an elipses. This is strictly for aesthetic/human- readability purposes.
Parameters:
{String} path
The file path needing to be shortended.
{Number} max Optional
The maximum number of slashes to include.
Returns:
{String} The normalized/shortened file path.
